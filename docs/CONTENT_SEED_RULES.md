# Content Seed Rules

These rules define how to add Content Library seed files during the mock/template stage.

## Adding A Spot

1. Add a JSON file under `backend/app/content_library/spots/`.
2. File name must match the `slug`, for example `kiyomizu.json`.
3. Include all shared fields: `id`, `slug`, `title`, `subtitle`, `description`, `story`, `tags`, `nearby`, `foods`, `hotels`, `source`, `language`.
4. Keep the file UTF-8 encoded.

## Naming Rules

- `id`: prefix by content type, for example `spot_kiyomizu`.
- `slug`: lowercase ASCII only, with `_` between words.
- `title`: user-facing Chinese name.
- `subtitle`: location or category, for example `京都 · 东山区`.

## Multilingual Rules

- Current language is `zh`.
- Do not mix multiple languages in one seed file.
- Future language variants must preserve the same slug and change only localized fields.

## Content Length

- `description`: 1-3 short sentences.
- `story`: 2-5 short sentences for AnswerCard display.
- `tags`: 3-8 tags.
- `nearby`, `foods`, `hotels`: keep each item concise.

## Content Type Rules

- `content_type` must match the folder category: `spot`, `city`, `food`, `route`, `culture`, or `hotel`.
- The value is used by the unified content index and repository lookup.
- Do not translate `content_type`.

## Aliases Rules

- `aliases` should include common user wording and short query phrases.
- Keep aliases in the same language as the seed file.
- Add aliases for likely search terms, for example `京都住宿`, `京都酒店`, `京都站酒店`.
- Do not use aliases for unrelated recommendation keywords.

## Priority Rules

- `priority` is an integer tie-breaker for ranking.
- Recommended range: 1-100.
- Use higher values for canonical pages such as city overview, classic spots, or primary hotel areas.
- Do not use priority to force unrelated content above a better textual match.

## Top-K Recommendation Rules

- Seed tags should help related content appear in top-k candidates.
- City pages can share tags with spots, foods, routes, and hotels.
- The primary card should match the strongest title or alias; related candidates can match tags.
- Keep tags focused so top-k results remain explainable.

## Prohibited Markers

Seed content must not claim to be generated by realtime AI. Do not write:

- `AI generated`
- `由 AI 生成`
- `token fallback`
- `模型不可用`
- `系统降级`

The system may use templates or AI later, but public content should remain user-facing and source-neutral.

# JAG-R024 Smart Regeneration UX Guide

## 原则

智能优化应该像旅行助手在帮用户减负，而不是像系统在报警。

## 推荐表达

- 已调整为更适合实际旅行的路线。
- 已减少跨城移动带来的疲劳。
- 已降低 Day 2 的停留密度。
- 已保留核心体验，同时增加休息余地。

## 避免表达

- `adaptation_count=3`
- `before_feasibility=overloaded`
- `fatigue_reduction=reduced`
- `transition_reduction=3`

这些字段只进入 debug/operator metadata，不进入 public UI。

## One-click Optimize

按钮文案使用：

- 智能优化路线

点击后应直接应用优化后的 timeline，并给出自然反馈：

- 已调整为更适合实际旅行的路线。
- 已智能优化路线。

## 降低焦虑

不要说“你的路线不合理”。应说：

- 这条路线偏满，我先帮你减少一些赶场。
- 跨城较多，我先把移动日安排得更轻松。

## AI Companion 感

建议卡应像助理建议：

- 自动优化建议
- 减少当天停留点
- 减少跨城疲劳
- 优化住宿区域

不要像后台规则命中列表。
